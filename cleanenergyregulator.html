Government Website Redesign:

Clean Energy Regulator

Client

Clean Energy Regulator

Role

UX/UI Designer

Timeline

3 weeks

Tools

Adobe XD, Photoshop, Miro

The Problem

The Clean Energy Regulator website is intended to be a trusted and informatory website. There are however, a number of issues that users face when using the site. Overwhelming navigations, messy layouts and unnecessarily complex steps required for users to achieve their goals.

The Solution

The website will be redesigned to a sophisticated, and modern level, with our users needs at the forefront of our focus. Navigations will become cleaner and streamlined, the homepage in particular will be more approachable, and the website less intimidating.

Website Analysis

  • Step one:

    Analysing the website for heuristic and accessibility issues.

  • Step two:

    Create a proto persona of a typical user of the Clean Energy Regulator website.

  • Step three:

    Use the proto persona to guide the user testing, analysing how a range of users respond to the website.

Heuristic and Accessibility Evaluation

  • The website was examined for violations of design heuristics and accessibility issues.

  • The standout issues:

    • Several failures of colour contrast

    • Confusing and excessive information on the homepage

    • Too many top navigation options

    • Pixelated logos and icons

    • Homepage feels confused

Proto Persona

  • Shaun is a corporate board member of a large Australian energy company.

    Shaun is tasked with visiting the Clean Energy Regulator website to gather knowledge about the Quarterly Carbon Market Report, Carbon Credits, and current Carbon Abatement Projects.

User Testing

  • Three user tests were conducted with participants between the ages of 25 and 65.

    The tests focused on the users ability to identify and use the navigation components on the site.
    Pain points and cosmetic issues were identified for future iterations.

2x2 Matrix

  • After analysing the user test results, a 2x2 matrix was created to evaluate and prioritise the user vs. Clean Energy Regulator's needs.

    Prioritised issues:

    • Homepage doesn't feel like a "homepage"

    • Gap between presented information and user understanding its meaning

    • Poorly structured information architecture

    • Confusing layout and use of navigations

Information Architecture

  • Stocktake was taken on the websites structure and current sitmap.

    Every page was written down on a "card" and sorted into similar, relevant categories.

    This strategy is called card sorting and is particularly useful in devising a newer, strategic sitemap.

Sitemap

  • Using card sorting, a new sitemap was constructed, offering several benefits for the user.

    Benefits:

    • Visually cleaner

    • Streamlined navigation

    • Cards sorted into alphabetical order

    • Removal and relocation of unnecessary cards

Low to Mid Fidelity Wireframing

  • Low to mid fidelity wireframing offers the opportunity to design the layout of each page, at minimal cost and time.


  • Wireframing is done for the desktop and mobile versions of the site.


    The benefits of low to mid fidelity wireframing is that any serious changes that need to be made can be done so without wasting substantial resources.

High Fidelity Prototyping

  • Once happy with the layout and structure of the website pages a high fidelity prototype was created.


    The high fidelity prototype uses the same font family and many of the same colours of the current website to keep it consistent with those familiar with the existing site.


    Navigations have been kept in the same location. However, the options have been simplified and alphabetised in the drop downs.
